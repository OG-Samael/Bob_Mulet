# Package Manager Guide

This project supports multiple package managers: **pnpm** (recommended), **npm**, and **yarn**.

## Quick Start

### Using pnpm (Recommended) ⭐
```bash
pnpm install
pnpm dev
```

### Using npm
```bash
npm install
npm run dev
```

### Using yarn
```bash
yarn install
yarn dev
```

## Configuration

The `.npmrc` file contains configurations for all three package managers:

### pnpm-specific:
- `auto-install-peers=true` - Automatically install peer dependencies
- `shamefully-hoist=true` - Hoist all dependencies to root node_modules
- `node-linker=hoisted` - Use hoisted node_modules structure

### npm-specific:
- `legacy-peer-deps=false` - Use npm v7+ peer dependency resolution
- `engine-strict=false` - Don't fail on engine version mismatches

### Shared:
- `strict-peer-dependencies=false` - Don't fail on peer dependency issues
- `save-exact=false` - Use version ranges (^) when installing

## Lock Files

Each package manager uses its own lock file:
- **pnpm**: `pnpm-lock.yaml` (currently used)
- **npm**: `package-lock.json`
- **yarn**: `yarn.lock`

⚠️ Only commit the lock file for your chosen package manager.

## Switching Package Managers

### From pnpm to npm:
```bash
Remove-Item pnpm-lock.yaml
npm install
```

### From pnpm to yarn:
```bash
Remove-Item pnpm-lock.yaml
yarn install
```

### From npm to pnpm:
```bash
Remove-Item package-lock.json
pnpm install
```

## Why pnpm is Recommended

✅ **Faster** - Uses hard links and symlinks
✅ **Disk efficient** - Shared global store
✅ **Stricter** - Better dependency isolation
✅ **Monorepo friendly** - Built-in workspace support

## Common Commands

| Task | pnpm | npm | yarn |
|------|------|-----|------|
| Install | `pnpm install` | `npm install` | `yarn` |
| Add package | `pnpm add <pkg>` | `npm install <pkg>` | `yarn add <pkg>` |
| Remove package | `pnpm remove <pkg>` | `npm uninstall <pkg>` | `yarn remove <pkg>` |
| Run script | `pnpm <script>` | `npm run <script>` | `yarn <script>` |
| Dev server | `pnpm dev` | `npm run dev` | `yarn dev` |
| Build | `pnpm build` | `npm run build` | `yarn build` |
| Lint | `pnpm lint` | `npm run lint` | `yarn lint` |

## Security Audits

### Check for vulnerabilities:
```bash
pnpm audit          # pnpm
npm audit           # npm
yarn audit          # yarn
```

### Fix vulnerabilities:
```bash
pnpm audit --fix    # pnpm
npm audit fix       # npm
yarn upgrade        # yarn (upgrades to latest)
```

## Troubleshooting

### Clear cache and reinstall:
```bash
# pnpm
pnpm store prune
Remove-Item -Recurse node_modules, pnpm-lock.yaml
pnpm install

# npm
npm cache clean --force
Remove-Item -Recurse node_modules, package-lock.json
npm install

# yarn
yarn cache clean
Remove-Item -Recurse node_modules, yarn.lock
yarn install
```

### Conflicts between package managers:
If you see strange errors, make sure you're only using one package manager and have only one lock file:
```bash
# Check what lock files exist
Get-ChildItem *lock* -Name

# Delete unused lock files
Remove-Item pnpm-lock.yaml     # if using npm/yarn
Remove-Item package-lock.json  # if using pnpm/yarn
Remove-Item yarn.lock          # if using pnpm/npm
```
